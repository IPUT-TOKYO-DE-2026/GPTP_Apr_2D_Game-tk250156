Mouse Drift
## ゲームの内容
コンセプト
マウス操作特有の「滑らかさ」を活かした操作感で、どれだけ効率よくアイテムを回収し、最短ルートでゴールできるかに挑戦。
システム
マウスポインターを動かし、クリックをした個所に移動する操作性

コア・ルール:
タイムアタック: 基準タイム「20秒」から経過時間を引いた値がベーススコアとなる。
スピードブーストアイテム(緑〇)で速度増加(一つ取るたびに1.2倍)。
アイテム収集: ステージ上のスコアアイテム（黄色◇）を取得することで、ベーススコアに加算。
リスク要素:障害物(赤い△や赤い□)に接触すると即座にゲームオーバー。画面上下にある赤い枠に当たってもゲームオーバー。
スコア要素：スコアや経過時間は内部ではかられ、プレイヤーが見ることはできない。ゴールして初めてわかる設計。

プレイヤーから見えなくした理由
・ 視覚的な情報の整理（ノイズの除去）
プレイヤーは「操作」と「障害物の回避」に集中。
内部では毎フレーム（1秒間に60回以上）細かい小数点以下の計算が行っているため、その激しく動く数字をすべて画面に出すと、目が疲れてしまい、肝心のゲーム画面が見えにくくなる。
・「没入感」と「演出」
計算過程が見えてしまうと、プレイヤーはゲームではなく計算機を見ている気分になる可能性がある。
プレイ中
操作に集中させるため、余計な情報は出さない。
終了後（リザルト）
 溜めていた結果をパッと出すことで、「達成感」や「驚き」を与える。

リザルトシステム
ゴール時に「TIME」と「SCORE」を算出。
[R]キーでリトライ、[T]キーでタイトルへ戻るスムーズな遷移。
